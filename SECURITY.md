# Security policy

## Supported versions

Security fixes are applied to the current production version on `main`.

## Reporting a vulnerability

Please use GitHub private vulnerability reporting for the repository. Do not open a public issue for a vulnerability that could expose user data or enable code execution.

Include the affected browser, reproduction steps, impact, and any suggested mitigation. You should receive an acknowledgement within seven days.

ChromaFlow intentionally has no backend. Its primary security boundary is keeping source images and derived pixels inside the browser tab.
